WebKitGTK color-scheme bug on initial page load

When the user's preference is set for dark mode, a webpage that uses the color-scheme meta element renders with light text on light background, causing the page to appear blank. The page renders as expected upon reload.

To reproduce, open this page in Web/Epiphany in a new tab or window.

Reproduced on May 25, 2024 in Web 46 (WebKitGTK 2.44.1) and Epiphany Technology Preview 46.0-89-g1aca13c01+ (WebKitGTK 2.45.2). WebKit bug reference: 264996 — [GTK] `color-scheme: light dark` shows white text on white
